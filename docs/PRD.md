# PRD — Family English Coach (家庭英语学习系统)

## 1. 产品定位

面向家庭成员的英语辅助学习工具。用户上传教材截图（每张对应一个 Unit），系统自动提取单词和句子，构建学习体系，并驱动后续练习、计划和统计。

## 2. 目标用户

家庭成员（非专业英语学习者），需要简单直观的操作界面。

## 3. 核心用户流程

```
上传教材图片 → OCR 自动解析 → 确认/编辑单词库 → 开始练习 → 记录掌握状态 → 查看统计
```

## 4. 功能模块

### 4.1 单词管理

| 功能 | 说明 |
|---|---|
| OCR 解析 | 上传教材图片，自动识别英文单词、句子、中文释义 |
| 单词确认 | 用户可编辑/删除/补充 OCR 结果 |
| 标签系统 | ⭐ 收藏、🔥 高频、📚 考试重点、❌ 不再练习、✅ 已记忆 |
| 掌握状态 | 未学习 → 学习中 → 熟悉 → 永久记忆 |
| 显示策略 | 已永久记忆单词减少出现；高频/重点单词增加权重 |

**单词状态机：**

```
unlearned → learning → familiar → permanent
                         ↓
                     (可回退到 learning)
```

**标签与状态的独立性说明：**
- 标签是用户主动标记（收藏、高频等），影响出题权重
- 掌握状态是系统根据练习结果自动计算
- 两者独立运作，共同影响单词出现频率

### 4.2 学习计划

| 功能 | 说明 |
|---|---|
| 每日目标 | 设定每天学习/复习的单词数量 |
| 单元目标 | 选择要学习的 Unit 范围 |
| 截止日期 | 设定完成日期，系统自动拆分每日任务 |
| 自动规划 | 综合剩余时间、剩余单词量、掌握情况生成每日任务 |
| 新词+复习混合 | 每日任务包含新词学习和旧词复习 |

### 4.3 记忆模式

| 模式 | 说明 | 阶段 |
|---|---|---|
| 单词卡 | 英文正面/中文背面翻转 | MVP |
| 拼写练习 | 给中文，拼英文 | MVP |
| 选择题 | 给英文，选中文释义 | MVP |
| 听写 | 播放发音，写英文 | P1 |
| 场景对话 | AI 生成含目标词的对话 | P1 |
| 图片联想 | 显示教材原图辅助记忆 | P1 |
| AI 对话练习 | 与 AI 进行情景对话 | P2 |
| 跟读评分 | 语音识别 + 发音评分 | P2 |

### 4.4 数据统计

| 指标 | 说明 |
|---|---|
| 全局掌握率 | 所有单词的掌握状态分布 |
| Unit 掌握率 | 每个 Unit 的掌握状态分布 |
| 学习趋势 | 每日学习量、正确率曲线 |
| 连续学习天数 | streak 统计 |

**掌握状态颜色规范：**

| 状态 | 颜色 | Hex |
|---|---|---|
| unlearned | 灰色 | `#9CA3AF` |
| learning | 橙色 | `#F97316` |
| familiar | 蓝色 | `#3B82F6` |
| permanent | 绿色 | `#22C55E` |

## 5. MVP 范围

最小闭环，验证核心价值链：

```
图片上传 → OCR 解析 → 单词库生成 → 单词卡/拼写/选择题练习 → 掌握状态记录 → 基础统计
```

**MVP 不包含：** 学习计划自动规划、AI 场景对话、听写、语音、Redis 缓存。

## 6. 非功能需求

- AI 调用必须通过抽象层，支持模型热替换
- 所有接口遵循 RESTful 规范，统一响应格式
- 数据库变更通过 Migration 管理
- 前端操作流程不超过 3 步完成核心任务

## 7. 术语表

| 术语 | 含义 |
|---|---|
| Unit | 一个教材单元，对应一张上传图片 |
| Word | 一个学习条目，含英文、中文、所属 Unit |
| Tag | 用户标记，影响出题权重 |
| Mastery | 掌握状态，系统根据练习自动计算 |
| Practice | 一次练习行为（单词卡翻转、拼写、选择等） |
